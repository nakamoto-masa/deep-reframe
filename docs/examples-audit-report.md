# examples.md 品質監査レポート

Issue #4 タスク2: 現行examples.mdをeval基準でチェックし、問題点を特定

## チェック基準

references.mdの診断基準を使用:

1. **Phase判定** — After側にPhase 1-2シグナルワードが主要訴求軸として残っていないか
2. **問題パターンの残留** — 技術が主語、自己宣言型、形容詞依存等がAfter側に残っていないか
3. **変換のポイントの明確さ** — 生成時の方向性ガイドとして機能するか
4. **ドメインカバレッジ** — どの領域に偏っているか

## セクション別結果

### セクション1: 手法名→顧客価値

| # | After | 判定 | 理由 |
|---|---|---|---|
| 1-1 | 欲しい性能から設計を逆算する | PASS | Phase 3、顧客主語 |
| 1-2 | 設計者が思いつかない構造を自動で発見する | PASS | Phase 3（「発見」）、顧客主語 |
| 1-3 | 熱・振動・強度をまとめて評価し、設計のやり直しをなくす | 要注意 | 「なくす」はPhase 2寄り（効率化）。苦痛除去であってPhase 3的な不可能→可能ではない |

### セクション2: スペック数字→顧客にとっての意味

| # | After | 判定 | 理由 |

## 結論

- **品質面**: 大半の例はeval基準をパスするが、3例でPhase 1-2表現が残っている

- カバレッジの偏りが主要な問題。品質面のFAILは3例のみで軽微
